Operators

Bitwise operators are like aritmetric operators, except they allow you to modify data on bit-level. They are used to modify bits or bytes.

NOT

Flipping, or negating bits can be done using the NOT (~) operator. It litterally toggles all the bits.

~0b01 = 10
~0b11 = 00

AND

The AND (&) operator returns 1 for each bit position only if the corresponding bits of both operands are 1’s.

0b01 & 0b11 = 01
0b01 & 0b00 = 00

OR

The OR (|) operator returns 1’s if the corresponding bits of either the operands is 1.

0b10 | 0b01 = 11
0b00 | 0b00 = 00
0b11 | 0b10 = 11

XOR

Similar to the OR operator, the XOR (^) operator only returns 1’s if either of the corresponding bits of the operands is 1, but not if both are.

0b10 ^ 0b01 = 11
0b00 ^ 0b00 = 00
0b11 ^ 0b10 = 01

You could consider it a shortcut of the following code.

a = 0b11
b = 0b10
(a | b) & ~(a & b) = 01

Shift

Bit shifting is the act of shifting a set of bits to the left or the right. This is used for pretty much everything.

To shift bits to the left, use <<. Additional bits will be added on the right-hand side.

0b1001 << 2 = 100100

To shift bits the other way, use >>. This will discard the right-hand bits.

0b1001 >> 2 = 1001

When shifting bits to the right, notice the amount of bits decreases? A zero-fill right shift (>>>) adds bits on the left-hand side, so the amount of bits remain the same.

0b1001 >>> 2 = 001001
