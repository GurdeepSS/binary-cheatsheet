Background

Looking at your computer screen, what do you see? Text, colours, images, video’s and much more. Everything you see, everything you store on your computer, every button you press is a stream of one’s and zero’s. Why? How does it work?

Computers parts are able to store data via tiny components called flip-flops. They can be set to two states: “one” and “zero.” These circuits store a single bit. The value of a bit is merely the numeric representation of a flip-flop, 1 or 0.

Fun fact: many switches on electronic devices have a 1 and 0 printed on them, to indicate which side of the switch turns a device on and off.

Computers have a lot of these circuits we can use to store many bits, and we can group bits to store more complex values. If one bit can have two different states, a group of two bits can have four. A group of four bits can have up to sixteen states. The amount of different states grows exponentially for every bit that is added to a group. s = n2, where s is the number of states, and n the number of bits within the group.

The de-facto standard is to call a group of bits a byte, and it is assumed one byte consists of 8 bits. The more explicit term for an 8-bit group is octet.

Binary is the purest representation of data on computers. It just isn’t very readable, but we can make it so!

Representation

Many high-level programming languages have datatypes like integers, floats, strings, etc. These datatypes tell the computer what to do with binary data. Instead of modifying bits and bytes to add two integers, we can use an arithmetic operator, and the computer will know what to do at binary level. Datatypes also help determine the representation of the binary data.

Let’s have look at the representation of an 8-bit unsigned integer (uint). A uint is a positive integer starting from 0. Incrementing is very similar to our decimal system:

  1. Increment the least significant bit
  2. If least significant bit reached the maximum value, reset and increment the left-hand bit
// No bits set => zero
00000000 = 0
// Increment least significant bit
00000001 = 1
// Least significant bit reached max, so we reset it and increment left-hand bit
00000010 = 2
00000011 = 3
00101010 = 42
11111111 = 255

Understanding uint is quite valuable, as many programming languages use the datatype as default representation for binary data.

const answer = 0b00101010;
console.log(answer); // 42
